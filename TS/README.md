### 📝 기록 원칙 (5-Steps)
모든 트러블슈팅 기록은 아래의 흐름을 따르려고 노력합니다.

1. **Problem (현상):** 어떤 에러나 예외 상황이 발생했는가?
2. **Analysis (원인 분석):** 로그 확인, 디버깅 등을 통해 파악한 근본 원인은 무엇인가?
3. **Trials (시도):** 문제를 해결하기 위해 어떤 가설을 세우고 시도했는가?
4. **Solution (해결):** 최종적으로 어떻게 해결했는가? (코드 포함)
5. **Lessons Learned (배운 점):** 재발 방지를 위해 무엇을 배웠는가?
